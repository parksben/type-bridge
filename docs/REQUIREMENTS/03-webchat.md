# WebChat 渠道（§二 2.10）

> 与飞书/钉钉/企微不同，WebChat 是 **TypeBridge 自带** 的渠道：用户不用任何企业 IM 账号，桌面 App 在本机启动一个 HTTP + WebSocket 服务，同 WiFi 下的手机扫码即可发消息。**完全不依赖任何云端服务**。技术细节见 [TECH_DESIGN/12-webchat.md](../TECH_DESIGN/12-webchat.md)。

---

## 2.10 WebChat 渠道（本地局域网网页机器人）

### 2.10.1 总体目标

为没有企业 IM 账号 / 不想暴露 IM 凭据 / 仅想快速试用的个人用户提供"开箱即用的输入桥"。整体形态：

- 桌面 App `连接 TypeBridge → WebChat` tab 点"启动会话"，App 在本机起一个 HTTP server（固定 8723，占用则递增 fallback）
- UI 展示 **二维码（OTP 已嵌入 QR URL）+ 本机 WiFi 名称**；二维码周围有围绕式进度环显示剩余有效时间（无倒计时数字）
- 同 WiFi 下手机扫码 → 进入聊天页（SPA 静态资源从本机 server 提供）→ **自动完成握手**（OTP 从 URL 参数中自动提取，无需手动输入）
- 发文本 / 图片 → 通过 Socket.IO 实时送到桌面 → 注入到当前聚焦输入框
- 桌面端注入完成后回手机一条 ack（"已注入"或"失败原因"）

**核心前提**：手机和电脑必须在**同一个 WiFi / 局域网**。

WebChat 与飞书 / 钉钉 / 企微 **完全平等**：可以同时启用，消息进入同一个 FIFO 队列，历史与日志都带 `webchat` 渠道标签。

### 2.10.2 用户流程（happy path）

1. 桌面打开 TypeBridge → `连接 TypeBridge` → `WebChat` tab
2. idle 态：一条简短的操作提示 + "启动会话"按钮（**不展示 WiFi 提醒**，见 §2.10.6）
3. 点"**启动会话**" → App 启动本机 HTTP + Socket.IO server → 显示 QR（围绕式进度环，无倒计时文字），底部提示"用手机扫码即可连接"
4. QR 码编码 `http://<本机 LAN IP>:<port>/?s=<sessionId>&otp=<otp_plain>`（每次 OTP 轮换时 QR 同步更新）
5. 用户掏手机扫码 → 手机浏览器加载 SPA → **自动提取 URL 中的 OTP 并握手**，无需用户输入任何验证码
6. Socket.IO 本地校验 OTP → 签发 userToken 给手机 → 切换到聊天页
7. 用户在手机上发消息 → Socket.IO 送到桌面 → 进入 FIFO 队列 → 注入到当前聚焦输入框 → Socket.IO ack 回手机
8. 手机消息流上每条消息显示 `已注入` / `失败：<原因>` 状态徽标

**OTP 自动轮换**：60 秒进度环归零时，桌面**自动生成新的 6 位 OTP 并刷新 QR 码**（无需用户点任何按钮）。**已绑定的手机不受影响**（server 按 userToken 校验后续消息）。只有未完成握手的新手机需要重新扫码。轮换是纯更新 OTP + 重置倒计时，**不重启 server、不中断已有连接**。

**手机端过期处理**：若扫到已过期的 QR 码（OTP 在扫码与握手之间刚好换轮），手机端自动检测并提示"二维码已过期，请重新扫码"，**无需用户手动输入验证码**。

### 2.10.3 安全模型

- **OTP 嵌入 QR URL**：QR 码里同时含 `sessionId` 和 `otp`，手机扫码后自动握手，无需手动输入验证码；OTP 不向局域网外暴露（server 仅绑局域网 IP）
- **服务仅本地可达**：server 绑在局域网 IP，同 WiFi 外的用户无法访问
- **OTP 60 秒自动轮换**：已绑定手机继续用 userToken 发消息，不受影响
- **OTP 5 次错误锁定**：连错 5 次自动作废 OTP，桌面 UI 切换到"验证码已锁定"态，用户手动点"重置 OTP"按钮生成新 OTP 解锁（**不会踢掉已绑定的手机**）
- **多手机共享**：同一 sessionId 允许同 WiFi 内多台手机分别输 OTP 绑定，各自签发独立 userToken；消息都进同一 FIFO 队列
- **桌面 App 关闭即会话终止**：server 进程退出，所有 Socket.IO 连接断开
- **OTP 不明文存储**：桌面内存里保留 sha256 + 明文（QR URL 生成用），持久化层不写
- **无云端依赖**：没有任何数据经过线上

### 2.10.4 消息形态

| 类型 | 手机端输入方式 | 注入到桌面 | 备注 |
|------|--------------|----------|------|
| 文本 | 键盘 | 文本注入到聚焦输入框 | 与飞书 / 钉钉 / 企微完全一致 |
| 图片 | 拍照 / 相册（仅 JPEG/PNG） | 图片粘贴到聚焦输入框 | 客户端压缩到 ≤2MB；iOS HEIC 自动转 JPEG（系统级） |
| 语音 | 直接用手机输入法的麦克风按钮 | 识别后的文字按文本路径注入 | WebChat **不自研语音识别**，也**不放语音入口按钮 / 引导弹层** |

**单向 + ack**：消息只从手机 → 桌面；桌面只回"注入成功 / 失败"作为 ack 状态徽标，不发反向消息。

#### 2.10.4.1 控制键快捷面板（手机端）

WebChat 移动端在输入框上方提供一排"控制键"快捷按钮，让用户通过手机直接控制桌面键盘行为（不输入文本，只触发按键事件）。

**面板位置与开合**：
- 位置：输入栏（ComposerBar）正上方
- 默认状态：**收起**。"展开/收起"切换按钮**内嵌在输入栏右侧**（原语音按钮的位置）
- 切换按钮使用 lucide **`Keyboard`** 图标。激活态填充 accent 色 + 白色图标；未激活态使用普通底色 + 中性图标色
- 展开/收起状态记入 `localStorage`，下次进入会话沿用

**按键集合与排列顺序（v1）**：

按钮从左到右：**上 → 下 → 左 → 右 → 空格 → 回车 → 删除**

| 按键 | 行为 |
|------|------|
| 上 / 下 / 左 / 右 | 方向键（光标移动 / 列表选择） |
| 空格 Space | 在桌面焦点输入框按下 Space |
| 回车 Enter | 在桌面焦点输入框按下 Return |
| 删除 Backspace | 在桌面焦点输入框按下 Backspace |

**与文本消息的执行顺序**：控制键事件**与文本消息共用同一 FIFO 队列**，避免回车插在粘贴中间导致提前提交。

**安全**：server 侧维护按键白名单（仅 Enter / Backspace / Space / Arrow 系列），任何不在白名单的 key code 一律拒绝。**不允许** Cmd / Ctrl / Option / 字母数字等可能引发任意操作的键。

### 2.10.5 入口阻断

- **PC 浏览器**（Mac / Windows / Linux）：显示"请用手机访问"全屏引导页
- **IM 内置浏览器**（微信 / 钉钉 / 飞书 / QQ）：无拦截，直接进入正常会话流程

UA 检测在 SPA 内做客户端判断（HTTP 模式无 server component）。

### 2.10.6 桌面端配置 tab

`连接 TypeBridge → WebChat` tab 的 UI 结构：

**① 启动前（idle 状态）** — 只有一条纯提示 + 一个启动按钮，不显示 WiFi 提醒：

```
┌─────────────────────────────────────────┐
│ ⓘ 点「启动会话」后，App 会在本机起一个  │
│   局域网 HTTP 服务，生成二维码 + 6 位   │
│   OTP。同 WiFi 下手机扫码、输入 OTP 后  │
│   即可发消息。                          │
│                                         │
│ [▶ 启动会话]                            │
└─────────────────────────────────────────┘
```

**② 启动后（session live — pending / bound 共用同一视图）**：QR 和 OTP 在整个会话周期都保持可见，仅头部横条根据是否已有设备绑定切换：

**② -a 尚未有设备绑定时（pending 状态）**：

```
┌─────────────────────────────────────────────┐
│ ⓘ 扫码之前确保手机与本机连接到同一 WiFi     │
│                                             │
│ ┌─────────────┬─────────────────────────┐   │
│ │             │ ● OTP 验证码       [↻] │   │
│ │             │ ┌─────────────────────┐ │   │
│ │  [QR 160]   │ │    5 2 7 0 9 3      │ │   │
│ │             │ ├─────────────────────┤ │   │
│ │             │ │█████████████────────│ │   │← 底部进度条
│ │             │ └─────────────────────┘ │   │
│ │ 用手机扫码  │ 剩余 00:45               │   │
│ └─────────────┴─────────────────────────┘   │
│ 手机扫码后输入此 6 位 OTP 完成绑定。         │
│ [⏸ 停止]                                     │
└─────────────────────────────────────────────┘
```

**② -b 已有 ≥1 台设备绑定时（bound 状态）**：QR + OTP + 进度条不变，顶部横条换成"已连接"提示。

**③ 验证码锁定（expired 状态，OTP 5 次错）**：

```
┌─────────────────────────────────────────┐
│ ⚠ 验证码已锁定                          │
│ 连续 5 次 OTP 错误，当前验证码已锁定。  │
│ 点下方按钮生成新 OTP 解锁（已绑定设备   │
│ 不会受影响）。                          │
│                                         │
│ [⟳ 重置 OTP]                             │
└─────────────────────────────────────────┘
```

**设计要点（v0.7+）**：
- **横排 QR + OTP**：QR 从 208px 缩到 160px，腾出右侧放 OTP 列
- **OTP 单容器展示**：一个圆角 pill 里用 `font-mono` + 字符间距拉开
- **进度条融入容器**：底部 3px 装饰条，不再单独占一行
- **手动刷新按钮**：OTP label 行右侧 `RotateCw` 图标按钮，点击立刻原子替换 OTP + 重置倒计时
- **不再有"重启会话"按钮**，避免踢掉已绑定的手机

握手成功瞬间：桌面 App 自动隐藏菜单栏窗口 + 系统通知"WebChat 已连接，请把焦点切到目标输入框"。

### 2.10.7 边界情况

| 场景 | 行为 |
|------|------|
| 端口 8723 被占用 | 递增尝试 8724–8732，全占则 UI 报错 |
| 手机切换网络 | Socket.IO 自动重连；30s 未恢复显示"连接中断，请检查 WiFi" |
| 桌面 App 退出 | server 进程停止，手机端显示"桌面已关闭" |
| OTP 60 秒倒计时归零 | 自动轮换 OTP；已绑定设备不受影响 |
| OTP 5 次错误 | 锁定 OTP；手机端显示错误提示；桌面切到"验证码已锁定"态 |
| 同 sessionId 多台手机扫码 | 各自输正确 OTP 后都能连上，消息串行注入 |
| 用户在 PC 浏览器输入 URL | 渲染"请用手机访问"页面 |
| 用户用微信 / 钉钉浏览器扫码 | 正常进入握手页，不再拦截 |
| 手机想用语音 | 直接点手机键盘麦克风按钮，识别后的文字正常注入 |

### 2.10.8 历史消息 / 日志 / 状态条同步

- 历史消息 chip 筛选器：`[ 全部 N ] [ WebChat ] [ 飞书 ] [ 钉钉 ] [ 企微 ]`
- 历史卡片右上角渠道 tag：`Smartphone` icon + "WebChat" + TypeBridge 品牌橙
- 系统日志前缀：`[webchat]`
- 侧边栏连接状态：`WebChat ● 已连接 (N 台设备)`

### 2.10.9 分发 / 打包

- **前端 SPA** (`webchat-local/dist/`) 作为 Tauri resource 打包进 `.app`
- **本地 server** 由 Tauri Rust 进程内嵌 axum + socketioxide 实现，无外部二进制
- **无语音模型 / 无 WASM 引擎**：.app 体积不受 WebChat 影响
- 用户完全离线也能使用

### 2.10.10 隐私声明

- **消息不出本机**：所有文本 / 图片通过局域网 Socket.IO 传到桌面，不经过任何中继
- **无 ASR / 无模型**：语音识别由用户手机自带输入法完成
- **OTP 不持久化**：进程结束即消失；持久化层只存 sha256
- **无追踪 / 无分析 SDK**：前端不接入任何统计 / 分析 / 广告 SDK
- **开源可审计**：server 端完整实现在 `src-tauri/src/webchat_server.rs`
