# WebChat 渠道（§二 2.10）

> 与飞书/钉钉/企微不同，WebChat 是 **TypeBridge 自带** 的渠道：用户不用任何企业 IM 账号，桌面 App 在本机启动一个 HTTP + WebSocket 服务，同 WiFi 下的手机扫码即可发消息。**完全不依赖任何云端服务**。技术细节见 [TECH_DESIGN/12-webchat.md](../TECH_DESIGN/12-webchat.md)。

---

## 2.10 WebChat 渠道（本地局域网网页机器人）

### 2.10.1 总体目标

为没有企业 IM 账号 / 不想暴露 IM 凭据 / 仅想快速试用的个人用户提供"开箱即用的输入桥"。整体形态：

- 桌面 App `连接 TypeBridge → WebChat` tab 点"启动会话"，App 在本机起一个 HTTP server（固定 8723，占用则递增 fallback）
- UI 在**未连接**时展示**二维码 + 本机 WiFi 名称**作为主视觉；二维码长期稳定（无 OTP、无倒计时、无刷新动画）
- 同 WiFi 下手机扫码 → 进入聊天页（SPA 静态资源从本机 server 提供）→ **直接握手绑定**（不需要任何验证码）
- 桌面**已连接**时主视觉切换为「已连接」动效画面（脉冲圈 + 设备名 + 重置按钮），二维码退到次要位置或隐藏
- 手机端绑定成功后展示一段短动效（脉冲圈 → 对勾），再进入聊天主页
- 发文本 / 图片 → 通过 Socket.IO 实时送到桌面 → 注入到当前聚焦输入框
- 桌面端注入完成后回手机一条 ack（"已注入"或"失败原因"）

**核心前提**：手机和电脑必须在**同一个 WiFi / 局域网**（这也是安全模型的第一道防线）。

WebChat 与飞书 / 钉钉 / 企微 **完全平等**：可以同时启用，消息进入同一个 FIFO 队列，历史与日志都带 `webchat` 渠道标签。

### 2.10.2 用户流程（happy path）

1. 桌面打开 TypeBridge → `连接 TypeBridge` → `WebChat` tab
2. idle 态：一条简短的操作提示 + "启动会话"按钮
3. 点"**启动会话**" → App 启动本机 HTTP + Socket.IO server → 显示 QR（无倒计时、无进度环），底部提示"用手机扫码即可连接"
4. QR 码编码 `http://<本机 LAN IP>:<port>/?s=<sessionId>`（**不含 OTP**，sessionId 长期稳定）
5. 用户掏手机扫码 → 手机浏览器加载 SPA → **直接握手**，server 校验请求来自同子网 IP 后立即绑定
6. 桌面端实时收到 `phase: bound` 事件 → **主视觉切换为「已连接」动效画面**（脉冲圈 + 设备名 + 重置按钮）
7. 手机端绑定成功瞬间播放一段短动效（脉冲圈 → 对勾，~1.2s），然后跳到聊天主页
8. 用户在手机上发消息 → Socket.IO 送到桌面 → 进入 FIFO 队列 → 注入到当前聚焦输入框 → Socket.IO ack 回手机
9. 手机消息流上每条消息显示 `已注入` / `失败：<原因>` 状态徽标

**没有 OTP 轮换、没有验证码倒计时**：二维码内容（sessionId）从 App 首次启动开始就**长期稳定**，可以截图分享给"自己的另一台手机"，但只对同子网设备生效。

**断连回退**：手机离线 / 锁屏超时 / WiFi 断开 → 桌面端 `phase` 回到 `pending`，**主视觉自动切回二维码**，准备接受重连。

### 2.10.3 安全模型（v3 重设计）

去掉 OTP 后，由两道防线共同保证安全：

**① LAN-only 绑定**：
- server 在绑定阶段强校验请求来源 IP **必须与本机网卡 IP 在同一子网**（默认 /24）
- 不同子网的请求（包括 VPN、跨网段、外网） → 立即拒绝，返回 `OUT_OF_LAN`
- 二维码截图泄露到外网也连不上，因为外网请求过不了 LAN 校验

**② 一次绑定 + 显式重置**：
- sessionId 一旦 `bound`（已有 ≥1 台手机绑定），**拒绝新设备绑定**，返回 `ALREADY_BOUND`
- 桌面 UI 在「已连接」主视觉里提供"重置二维码"按钮 → 一键解绑当前设备 + 生成新 sessionId
- 这道防线保证即使二维码截图泄露给同子网内的他人（合租 / 共用 WiFi），也不会被偷偷接入

**其他安全约束**：
- **多手机共用**：sessionId 未绑定（`pending`）时，第一个扫码的手机绑定成功；后续扫码的手机走 `ALREADY_BOUND` 拒绝；如需多手机同时连，桌面端「重置二维码」生成新会话即可
- **桌面 App 关闭即会话终止**：server 进程退出，所有 Socket.IO 连接断开；sessionId 本身**保留在 settings.json**，下次启动复用（这是"二维码长期稳定"的实现基础）
- **手动重置入口**：除「已连接」状态下的"重置二维码"按钮外，桌面 WebChat 设置区也提供一个"高级 → 重置 sessionId"入口，覆盖极端情况下用户想换会话的需求
- **无云端依赖**：没有任何数据经过线上

### 2.10.4 消息形态

| 类型 | 手机端输入方式 | 注入到桌面 | 备注 |
|------|--------------|----------|------|
| 文本 | 键盘 | 文本注入到聚焦输入框 | 与飞书 / 钉钉 / 企微完全一致 |
| 图片 | 拍照 / 相册（仅 JPEG/PNG） | 图片粘贴到聚焦输入框 | 客户端压缩到 ≤2MB；iOS HEIC 自动转 JPEG（系统级） |
| 语音 | 直接用手机输入法的麦克风按钮 | 识别后的文字按文本路径注入 | WebChat **不自研语音识别**，也**不放语音入口按钮 / 引导弹层** |

**单向 + ack**：消息只从手机 → 桌面；桌面只回"注入成功 / 失败"作为 ack 状态徽标，不发反向消息。

#### 2.10.4.1 快捷键面板（手机端"键盘"Tab）

WebChat 移动端顶部 Tab 栏提供三个 Tab：**打字 / 触控 / 键盘**。"键盘"Tab 以整页形式展示快捷键面板，让用户通过手机直接控制桌面键盘行为（不输入文本，只触发按键 / 快捷键事件）。

**位置**：独立的顶部 Tab（`键盘` / `Keyboard`），与"打字"和"触控"并列，点击后整页展示。

**横滑分页（v0.2.3+）**：三组分类（截图 / 编辑 / 翻页·光标）改为**横向滑动切屏**展示，底部以圆点 indicator 指示当前页（类似 iOS 桌面分页点）。每屏顶部居中放一个小标题（淡色、不抢戏）。

| 分页 | 按键内容 |
|-----|---------|
| 截图 | 截窗口 / 截全屏 / 粘贴截图（卡片式大按钮，单屏 2+1 布局） |
| 编辑·输入 | 撤销 / 重做 / 换行 / 删除 / 清空 / 退出 / 全选 / 复制 / 剪切 / 粘贴 |
| 翻页·光标 | ↑↓←→（D-pad）+ 行首/行尾 + 上翻页/下翻页 + 文档首/文档尾 |

按钮尺寸大（截图按钮 96px、其他键盘按钮 ≥76px），方便手指点按。

**"文本"发送行为**：
- **仅发送**（默认）：文本注入到焦点输入框，**不触发**桌面的自动提交逻辑
- **发送并提交**：文本注入后额外执行一次桌面配置的提交组合键（等同桌面"自动提交"触发一次），由手机 `text` 事件携带 `submit: true` 字段控制

**与文本消息的执行顺序**：按键事件与文本消息共用同一 FIFO 队列，避免回车插在粘贴中间导致提前提交。

**安全**：server 侧维护两套白名单：
- `ALLOWED_KEY_CODES`：单键（Enter / Backspace / Escape / Space / Arrow 系列 / Home / End / PageUp / PageDown）
- `ALLOWED_COMBOS`：快捷键（Undo / Redo / SelectAll / Copy / Cut / Paste / DocTop / DocBottom / DesktopLeft / DesktopRight / MissionControl / AppExpose）

任何不在白名单的 code / combo 一律拒绝，**不入队**。

### 2.10.5 入口阻断

- **PC 浏览器**（Mac / Windows / Linux）：显示"请用手机访问"全屏引导页
- **IM 内置浏览器**（微信 / 钉钉 / 飞书 / QQ）：无拦截，直接进入正常会话流程

UA 检测在 SPA 内做客户端判断（HTTP 模式无 server component）。

### 2.10.6 桌面端配置 tab（v3 主视觉两态）

`连接 TypeBridge → WebChat` tab 的 UI 在 session live 状态下有**两个主视觉态**：

**① 启动前（idle 状态）** — 只有一条纯提示 + 一个启动按钮：

```
┌─────────────────────────────────────────┐
│ ⓘ 点「启动会话」后，App 会在本机起一个    │
│   局域网 HTTP 服务，生成二维码。同 WiFi  │
│   下手机扫码即可连接。                   │
│                                         │
│ [▶ 启动会话]                            │
└─────────────────────────────────────────┘
```

**② -a 已启动，未连接（pending 状态 — 二维码主视觉）**：

```
┌─────────────────────────────────────────────┐
│ ⓘ 扫码之前确保手机与本机连接到同一 WiFi     │
│                                             │
│         ┌─────────────────────┐             │
│         │                     │             │
│         │      [QR 240]       │             │
│         │                     │             │
│         └─────────────────────┘             │
│         用手机扫码即可连接                   │
│                                             │
│ [⏸ 停止]                                     │
└─────────────────────────────────────────────┘
```

**② -b 已连接（bound 状态 — 动效主视觉）**：

```
┌─────────────────────────────────────────────┐
│              ⭕ ⭕ ⭕                         │
│            脉冲圈动效                        │
│           ✓ 已连接                          │
│                                             │
│           iPhone-XXX                        │
│                                             │
│        [↻ 重置二维码]                        │
│                                             │
│ [⏸ 停止]                                     │
└─────────────────────────────────────────────┘
```

**两态切换语义**：
- `phase: pending → bound` → 主视觉从 QR 渐变为成功动效（200ms 淡入淡出）
- `phase: bound → pending`（手机断连 / 锁屏超时 / 桌面手动重置） → 主视觉自动切回 QR，准备重新接受扫码
- 不再有任何 OTP 倒计时 UI、OTP 锁定态、OTP 重置按钮

**重置二维码语义**：
- 在 bound 主视觉里点击 → 解绑当前手机 + 生成新 sessionId + 切回 QR 主视觉
- 用户场景：① 临时让朋友扫码用一下，用完想要回收 ② 觉得二维码可能被人偷拍过 ③ 想换一台手机

握手成功瞬间：桌面 App 自动隐藏菜单栏窗口 + 系统通知"WebChat 已连接，请把焦点切到目标输入框"（已有逻辑保留）。

### 2.10.7 边界情况

| 场景 | 行为 |
|------|------|
| 端口 8723 被占用 | 递增尝试 8724–8732，全占则 UI 报错 |
| 手机切换网络 | Socket.IO 自动重连；30s 未恢复显示"连接中断，请检查 WiFi" |
| 手机锁屏 / 切后台 | 前端 `visibilitychange` 监听，回前台时自动重连复用现有 binding（保活基础版） |
| 桌面 App 退出 | server 进程停止，手机端显示"桌面已关闭"；sessionId 落盘保留下次复用 |
| 同 sessionId 第二台手机扫码 | 返回 `ALREADY_BOUND`；手机端提示"该会话已绑定其他设备，请桌面端点'重置二维码'后再扫" |
| 跨子网扫码（VPN / WAN / 不同网段 WiFi） | 返回 `OUT_OF_LAN`；手机端提示"请确保手机与电脑连同一 WiFi" |
| 用户在 PC 浏览器输入 URL | 渲染"请用手机访问"页面 |
| 用户用微信 / 钉钉浏览器扫码 | 正常进入握手页，不再拦截 |
| 手机想用语音 | 直接点手机键盘麦克风按钮，识别后的文字正常注入 |
| 桌面切换 WiFi 网段 | server 可能继续监听旧 IP（已知 macOS 现象） → 用户重启 App 让其绑当前 IP |

### 2.10.8 历史消息 / 日志 / 状态条同步

- 历史消息 chip 筛选器：`[ 全部 N ] [ WebChat ] [ 飞书 ] [ 钉钉 ] [ 企微 ]`
- 历史卡片右上角渠道 tag：`Smartphone` icon + "WebChat" + TypeBridge 品牌橙
- 系统日志前缀：`[webchat]`
- 侧边栏连接状态：`WebChat ● 已连接 (N 台设备)`

### 2.10.9 分发 / 打包

- **前端 SPA** (`webchat-local/dist/`) 作为 Tauri resource 打包进 `.app`
- **本地 server** 由 Tauri Rust 进程内嵌 axum + socketioxide 实现，无外部二进制
- **无语音模型 / 无 WASM 引擎**：.app 体积不受 WebChat 影响
- 用户完全离线也能使用

### 2.10.10 隐私声明

- **消息不出本机**：所有文本 / 图片通过局域网 Socket.IO 传到桌面，不经过任何中继
- **无 ASR / 无模型**：语音识别由用户手机自带输入法完成
- **sessionId 持久化但仅落盘到 settings.json**：用户清空设置即可重置
- **无追踪 / 无分析 SDK**：前端不接入任何统计 / 分析 / 广告 SDK
- **开源可审计**：server 端完整实现在 `src-tauri/src/webchat_server.rs`
